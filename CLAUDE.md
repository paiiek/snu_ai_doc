# 프로젝트: slides2manuscript

전역 `~/.claude/CLAUDE.md`(oh-my-claudecode)의 원칙을 따르되, 이 저장소에서는
아래 프로젝트 규칙이 우선한다.

## 무엇을 만드는가

조교가 받은 **강의 슬라이드 PDF**를 **줄글 강의 원고(.docx)**로 변환하는
독립 실행 파이썬 CLI. 발표 대본이 아니라 사람이 쓴 듯한 한국어 산문 원고를 만든다.
다른 조교들에게 폴더째 배포해 쓰는 것이 목표다(별도 도구 의존성 없이).

## 절대 원칙 (출력물 품질)

1. **분량은 반드시 채운다 — 제1원칙.** 과제 기준별 최소 쪽수는 무조건 보장한다.
   - 내부 1강: 35쪽 / 내부 2강: 70쪽
   - 외부 1강: 17쪽 / 외부 2강: 34쪽
   `pages × chars_per_page`로 목표 글자 수를 잡고, 부족하면 수렴 루프가 모든 장을
   슬라이드 근거 안에서 확장해 최소 쪽수를 채운다. **다른 어떤 원칙보다 이 원칙이
   우선한다.** 슬라이드가 이미 다룬 논점에 대해 부연·재서술·예시 확장·맥락 보강·
   예상 질문 답변 등을 모두 활용해 반드시 분량을 채운다. 슬라이드 원문 대비 확장
   배율이 커질수록 늘어짐이 강해질 수 있으므로 결과물은 사람이 검토·보완하지만,
   검토 필요성이 분량 미달의 이유가 되지는 못한다.
2. **부연 확장이 우선 — 근거 있는 외부 지식도 허용.** 슬라이드가 다룬 개념·주장은
   물론이고, 그 개념을 이해하는 데에 도움이 되는 **일반적으로 확립된 외부 지식**
   (표준 용어의 정의, 널리 알려진 배경 사례, 확실한 근거가 있는 인접 개념, 표준
   교재 수준의 부연)도 자유롭게 추가해 분량을 채울 수 있다. 필요할 경우 유사 사례
   반복 서술이나 개념의 여러 각도에서의 재정립 같은 중복·장황함도 허용된다.
   원칙 1(분량 보장)을 만족하기 위해서라면 이 자유를 적극 활용한다.
   여전히 하지 말아야 할 것: **확인되지 않은 사실·수치·인용·출처 지어내기**,
   **특정 슬라이드에 없는 특정 인물 발언의 재구성**, **존재하지 않는 논문·URL 만들기**.
   즉 "확실한 근거"의 범위 안에서만 외부 지식을 결합한다. 참고 자료는
   `--ref-file`/`--ref-url` 로 명시적으로 주입된 것에 한한다.
3. **줄글 산문만.** 글머리표·번호목록·표·마크다운 기호·이모지 금지.
   "다음과 같다", "아래 그림" 같은 슬라이드 지시 표현도 쓰지 않는다.
4. **AI 티 제거.** 정형화된 도입/마무리 문구, '결론적으로/요약하자면' 남발,
   모든 문단을 접속사로 여는 습관, 같은 구조·길이 문장 반복 금지. 문장 길이를 변주한다.
   강의자가 직접 쓴 평서형('-이다/-한다') 학술 강의체. 구어 추임새('자', '여러분') 금지.
5. **드라이한 서식.** 콘진원 원고료 책정표 규격을 따른다: **신명조 12pt · 35줄/쪽 ·
   상하 15mm · 좌우 20mm · 머리말/꼬리말 15mm**. 본문 글꼴은 하나만 쓰고, 장 제목만
   살짝 키운다. 색·표·테두리·장식 없음. 35줄/쪽을 지키기 위해 문단 사이 추가 여백은
   두지 않는다(space_after = 0).

위 원칙 가운데 프롬프트에 직접 실리는 부분(1·2·3·4번)의 실제 지침 문구는
`slides2manuscript/prompts.py`의 `STYLE_RULES`에 있다. 원고 톤을 바꿀 일이
생기면 코드 흩뿌리지 말고 거기서만 고친다.

## 구조

```
make_manuscript.py            # 진입 스크립트 (python make_manuscript.py 강의.pdf)
slides2manuscript/
  cli.py          # argparse, 파이프라인 오케스트레이션, 분량 질문(_resolve_volume),
                  #   장 수 자동 산정(_auto_sections), 분량 수렴(_expand/_condense), 키 설정
  extract.py      # PyMuPDF로 슬라이드별 텍스트 추출 + 이미지PDF 감지
  refs.py         # (--ref-file/--ref-url) 강사 홈페이지·수업자료 등 참고 자료 로드
                  #   → 프롬프트 뒤에 [참고 자료] 블록으로 주입. 슬라이드 부족분 보강용
  vision.py       # (--vision) 슬라이드를 이미지로 렌더링 → 비전 모델로 내용 보강(병렬)
  llm.py          # provider 추상화(anthropic/openai): chat / vision_chat
  prompts.py      # 모든 프롬프트(아웃라인/장 본문/확장·압축) + STYLE_RULES  ← 톤의 단일 출처
  generate.py     # design_outline → write_section, 분량 배분
  docx_writer.py  # 드라이 docx 출력(한글 글꼴 강제, A4/여백/줄간격 고정)
                  #   본문 폰트: 나눔명조 있으면 "나눔명조"(한글명, macOS LO는 이 이름만 매칭),
                  #   없으면 OS별 명조체 폴백(macOS→AppleMyungjo, Windows→바탕)
  pdf_export.py   # docx → pdf 자동 변환. LibreOffice(soffice) 우선, 없으면 docx2pdf(=Word)
  font_setup.py   # (--install-font) 나눔명조(OFL) 3종을 사용자 폰트 폴더에 다운로드·설치
```

파이프라인: 추출 → (선택)비전 보강 → 장 구성 설계(1콜) → 장별 본문 생성(장 수만큼 콜)
→ 분량 수렴 보정 → docx 저장 → (기본 동작) pdf 변환.

## 프로덕션 워크플로 — 분량·확장·검증 사이클

원칙 1(분량 무조건 채움)을 실무에서 안정적으로 만족시키기 위한 표준 사이클.
이 사이클을 지키면 원고가 목표 쪽수에 확실히 도달하고, 부연으로 넣은 사실도
검증된 채로 남는다.

1. **초안 생성**. 파이프라인을 돌려 초안 docx/pdf 생성. `chars_per_page` 실측 후
   목표 대비 현재 쪽수를 계산한다(예: 서봉원 v1은 57k자·35쪽으로 70쪽 목표의 절반).
2. **분량 진단**. 목표 쪽수 대비 부족량을 자·쪽 단위로 명시. 부족량이 크면
   3~4번을 여러 배치로 나눠 반복한다.
3. **부연 확장** (원칙 2에 근거). 각 장의 body 뒤에 부연 문단을 추가한다.
   재료의 우선순위:
   - (a) 슬라이드가 이미 다룬 논점의 배경·맥락·함의·예상 질문 답변
   - (b) `--ref-file`/`--ref-url`로 넣은 강사 자료·홈페이지 텍스트
   - (c) 확실한 근거가 있는 외부 표준 지식(용어 정의, 널리 알려진 배경, 표준 교재 수준 부연)
   (a) → (b) → (c) 순으로 시도하고, (c)를 쓸 때는 반드시 4번 검증을 붙인다.
4. **웹 검증** (외부 지식을 쓴 경우). 새로 넣은 사실적 주장(연도·인물·수치·논문 ID
   등)을 웹 검색으로 확인한다. 오류 발견 시 원고 수정. 슬라이드 근거이거나 확립된
   표준 지식은 검증 생략 가능하지만, 애매한 것은 무조건 검증한다. 검증 통과한 항목
   목록을 사용자에게 표로 보고하는 것이 표준이다.
5. **분량 재측정**. 다시 docx/pdf 생성해 실측 쪽수 확인. 목표 미달이면 3~4로 복귀.
   목표 도달이면 6번.
6. **최종 저장 및 커밋**. 각 강의 폴더에 `<강사이름>_원고.docx`와 `.pdf`로 저장.
   개인 원고 자체는 `.gitignore`로 리포지토리 커밋에서 제외되지만, 도구 개선 사항
   (프롬프트·CLI·규격 조정 등)은 커밋 후 푸시.

이 사이클은 지금은 사람이(또는 이 세션의 Claude가) 수동으로 오케스트레이션하지만,
반복적인 배치·확장·검증 단계는 향후 스크립트나 서브에이전트로 자동화할 여지가 있다.
현재로서는 사용자가 `--ref-file`로 강사 홈페이지나 관련 논문의 텍스트를 미리
넣어 두면 3번 단계가 훨씬 안정적이다.

## 작업 규칙

- **유료 API 호출 주의.** `generate.py`의 실호출(`messages.create`)은 사용자 키로
  과금된다. 사용자가 명시적으로 요청하지 않는 한 실제 PDF로 전체 파이프라인을
  돌리지 마라. 로직 검증은 API를 타지 않는 경로(extract/docx/fallback/budget)로 한다.
- **텍스트 기반 PDF가 기본.** 텍스트가 적은 이미지/스캔 PDF는 기본적으로 막고(`--force`),
  `--vision`이면 슬라이드를 이미지로 읽어 처리한다(`vision.py`, 슬라이드당 비전 1콜 추가).
- 기본 모델은 `claude-opus-4-8`(자연스러운 한국어 우선). 비용 옵션은 `--model claude-sonnet-4-6`.
- 의존성은 최소로 유지한다(pymupdf, anthropic, openai, python-docx). 배포 단순성이 중요하다.
- 한국어 주석/메시지를 유지한다. 사용자와 다른 조교가 읽는다.

## 테스트

API 없이 확인 가능한 부분만으로 회귀를 잡는다.

```bash
pip install -r requirements.txt
python make_manuscript.py --help                 # CLI 파싱
# 추출/ docx / 폴백 / 분량배분 경로는 임시 PDF를 만들어 단위 확인 (scratchpad)
```

실제 원고 품질(줄글다움, AI 티, 충실성)은 사용자가 본인 키로 1회 돌려 육안 검토한다.
