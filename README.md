# 강의 슬라이드 → 강의 원고 변환기 (slides2manuscript)

교수님 **슬라이드 PDF**를 넣으면, 발표 대본이 아니라 **사람이 직접 쓴 듯한 줄글 강의 원고(.docx)**
로 바꿔 주는 프로그램입니다.

> 강사분들의 발표 슬라이드를 바탕으로 강의 원고를 제출해야 하는 조교분들의 수고를 덜기
> 위해 만들었습니다. 문의는 **기술분과 조교 백승렬**(010-9265-8369, paik402@snu.ac.kr)으로
> 주시면 됩니다.

- 입력: **슬라이드는 반드시 PDF로 넣어야 합니다** (텍스트 기반 PDF, 파워포인트·키노트에서 내보낸 것)
- 출력: A4 워드 문서(.docx). 서식은 콘진원 원고료 책정표 규격을 따릅니다
  (**신명조 12pt / 35줄/쪽 / 상하 15mm · 좌우 20mm · 머리말·꼬리말 15mm**)
- 특징: 슬라이드 내용에 충실 · 개조식(글머리표) 없이 줄글 · AI 티 안 나는 문장

---

## 설치·사용 순서

처음 한 번만 설치와 키 발급이 필요하고(약 10~15분), 그다음부터는 명령어 한 줄이면 됩니다.

1. 파이썬(Python) 설치
2. 이 프로그램 내려받기
3. 라이브러리 설치
4. API 키 발급
5. 키를 프로그램에 알려주기 (실행하면 물어봄 — 가장 쉬움)
6. 실행

> API 키는 AI(클로드 또는 GPT)를 호출할 때 쓰는 인증 키입니다.
> 본인이 발급받아야 하며, 사용한 만큼 요금이 부과됩니다.

## 1단계. 파이썬 설치

### 맥(Mac)
1. https://www.python.org/downloads/ 접속 → 노란 **Download Python** 버튼 클릭.
2. 내려받은 `.pkg` 파일을 더블클릭 → "계속 / 동의 / 설치" 를 끝까지 누름.

### 윈도우(Windows)
1. https://www.python.org/downloads/ 접속 → **Download Python** 버튼 클릭.
2. 내려받은 설치 파일 실행.
3. ⚠️ **중요**: 첫 화면 맨 아래 **"Add Python to PATH"** 체크박스를 반드시 켠 뒤
   **Install Now** 클릭.

설치 확인(선택): 터미널(맥) 또는 명령 프롬프트(윈도우)를 열고 아래를 입력해
버전 숫자가 나오면 성공입니다.
```
python3 --version
```
(윈도우에서 안 되면 `python --version` 으로 시도)

> **터미널/명령 프롬프트 여는 법**
> - 맥: `Command(⌘) + 스페이스` → "터미널" 입력 → 엔터
> - 윈도우: 시작 메뉴에서 "cmd" 또는 "PowerShell" 검색 → 실행

## 4단계. API 키 발급받기

**클로드(Anthropic)와 GPT(OpenAI) 중 하나만** 있으면 됩니다. 가진 계정을 쓰세요.

### 클로드(Anthropic) 키 — 기본 권장
1. https://console.anthropic.com 접속 → 가입/로그인.
2. 왼쪽 메뉴 **Billing** 에서 결제수단 등록(선불 충전식. 소액 충전 가능).
3. 왼쪽 메뉴 **API Keys** → **Create Key** → 이름 아무거나 → 생성.
4. 나온 키(`sk-ant-...`)를 복사해 둡니다. **이 화면을 닫으면 다시 못 보니 꼭 복사.**

### GPT(OpenAI) 키 — 클로드 안 쓰는 경우
1. https://platform.openai.com/api-keys 접속 → 로그인.
2. 결제수단 등록(Billing) 후 **Create new secret key** → 키(`sk-...`) 복사.

> 💰 **비용 안내**: 강의 한 편(슬라이드 수십 장 → 35쪽 이상 원고) 변환에 모델 호출
> 요금이 듭니다. 정확한 금액은 모델·분량에 따라 다릅니다. 비용을 아끼려면
> 6단계에서 `--model claude-sonnet-4-6`(클로드) 같은 저렴한 모델 옵션을 쓰세요.

## 5단계. 키를 프로그램에 알려주기

방법이 두 가지입니다. **방법 A(자동 입력)가 가장 쉽습니다.**

### 방법 A — 실행하면 물어보는 대로 붙여넣기 (추천, 특히 맥)
키 파일을 직접 만들 필요가 없습니다. 그냥 6단계처럼 프로그램을 실행하면,
키가 없을 때 이렇게 물어봅니다.
```
API 키가 아직 설정되어 있지 않습니다. 지금 입력하면 key.txt 에 저장해 둡니다.
  1) Anthropic(클로드) 키 사용   2) OpenAI(GPT) 키 사용
번호 입력(1 또는 2):
```
- 번호(1 또는 2)를 누르고 엔터.
- 4단계에서 복사해 둔 키를 **붙여넣고**(맥은 `⌘+V`, 윈도우는 마우스 우클릭) 엔터.
- 끝입니다. 다음부터는 `key.txt` 에 저장돼 있어 다시 안 물어봅니다.

> 붙여넣을 때 키 글자가 화면에 안 보여도 정상입니다. 그대로 엔터 치세요.

### 방법 B — key.txt 파일을 직접 만들기
1. 프로그램 폴더 안의 **`key.txt.example`** 파일을 복사한 뒤, 사본 이름을
   **`key.txt`** 로 바꿉니다.
2. `key.txt` 를 **텍스트편집기(맥)/메모장(윈도우)**으로 엽니다.
3. 본인 키에 맞는 줄의 값만 바꿔 저장합니다.
   - 클로드: `ANTHROPIC_API_KEY=sk-ant-여기에-본인-키`
   - GPT: 클로드 줄은 지우고 `OPENAI_API_KEY=sk-여기에-본인-키` 줄을 쓰세요.

> 🍎 **맥 사용자 주의 (중요)**
> - 맥 Finder는 `.env` 처럼 **점(.)으로 시작하는 파일을 못 만듭니다**(경고창이 뜸).
>   그래서 점이 없는 **`key.txt`** 를 씁니다. (예전 안내의 `.env` 는 신경 쓰지 마세요.)
> - 파일을 복사할 때 `key.txt.example` 이 **안 보이면**, Finder에서 `⌘+Shift+.`(점) 을
>   누르면 숨김 파일이 보입니다. 보통은 `key.txt.example` 은 점이 없어 그대로 보입니다.
> - "확장자를 .txt 에서 바꾸겠습니까?" 같은 창이 떠도 그대로 두면 됩니다(이름은 key.txt).

> 🔒 `key.txt` 는 **본인 컴퓨터에만** 있고 GitHub에 올라가지 않습니다(자동 차단됨).
> 키는 비밀번호이니 누구에게도 보내지 마세요.

## 6단계. 실행하기

1. 변환할 **슬라이드 PDF를 프로그램 폴더 안에 복사**해 넣습니다.
2. 터미널에서(3단계처럼 폴더로 이동한 상태에서) 아래를 입력합니다.
   따옴표 안에 **본인 PDF 파일 이름**을 넣으세요:
   ```
   python3 make_manuscript.py "강의1.pdf"
   ```
   (윈도우에서 `python3` 가 안 되면 `python make_manuscript.py "강의1.pdf"`)
3. 실행하면 **분량 기준을 물어봅니다.** 과제에 맞게 번호를 누르세요(그냥 엔터=기본 1강 내부).
   ```
   원고 분량 기준을 선택하세요. (그냥 엔터를 누르면 기본값)
     강의 수 —  1) 1강(기본)   2) 2강 연속
     구분   —  1) 내부 35/70장(기본)   2) 외부 17/34장
   ```
   | 선택 | 보장 최소 분량 |
   |---|---|
   | 1강 · 내부 | 35쪽 이상 |
   | 1강 · 외부 | 17쪽 이상 |
   | 2강 · 내부 | 70쪽 이상 |
   | 2강 · 외부 | 34쪽 이상 |
4. 진행 상황이 화면에 표시되고(몇 분 소요), 끝나면 같은 폴더에
   **`강의1_원고.docx`** 가 생깁니다. 워드/한글로 열어 확인하세요.

### 자주 쓰는 옵션
```
python3 make_manuscript.py "강의1.pdf" -t "1주차: 인공지능 개요" -o "1주차_원고.docx"
```
| 옵션 | 뜻 | 예시 |
|---|---|---|
| `-t` | 원고 맨 위 제목 | `-t "1주차 강의"` |
| `-o` | 저장할 파일 이름 | `-o "1주차_원고.docx"` |
| `--pages` | 목표 분량(쪽). 기본 38 | `--pages 55` |
| `--min-pages` | 최소 보장 분량(쪽) | `--min-pages 35` |
| `--model` | 사용할 AI 모델(비용 절감용) | `--model claude-sonnet-4-6` |
| `--provider` | `anthropic` 또는 `openai` 강제 지정 | `--provider openai` |
| `--vision` | 슬라이드 그림·도표·수식까지 읽어 반영 | `--vision` |
| `--ref-file` | 참고 자료 파일(.txt/.md/.pdf/.docx). 여러 개 지정 가능 | `--ref-file 강사소개.txt 커리큘럼.pdf` |
| `--ref-url` | 참고 자료 URL. 강사 홈페이지·전시 소개 등의 본문 텍스트를 근거로 추가 | `--ref-url https://.../artist-bio` |

### 그림·도표가 많은 슬라이드라면 — `--vision`
글자는 적고 **그림·도표·그래프·수식 위주**인 강의는, 텍스트만으로는 내용이 부족해
원고가 빈약하거나 억지로 늘어납니다. 이때 `--vision` 을 붙이면 각 슬라이드를
**이미지로 읽어** 그림·도표·수식의 내용까지 원고에 반영합니다. 이미지만 있는
(텍스트가 전혀 없는) PDF도 처리할 수 있습니다.
```
python3 make_manuscript.py "강의.pdf" --vision
```
> 슬라이드 수만큼 읽기 호출이 추가되어 **시간과 비용이 늘어납니다.** 텍스트가 충분한
> 자료에는 굳이 쓰지 않아도 됩니다. 텍스트가 적은 자료에 특히 권장합니다.

### 슬라이드 밖 자료도 근거로 쓰고 싶다면 — `--ref-file`, `--ref-url`
예술 강사처럼 슬라이드가 **작품 이미지 위주**여서 텍스트가 부족한 경우, 강사의
**홈페이지 소개글, 아티스트 스테이트먼트, 수업 안내서, 커리큘럼 문서** 등을 함께
근거로 넣을 수 있습니다. 두 가지 방법이 있으며, **안정성은 파일 방식이 낫습니다.**

**방법 1 — 파일로 넣기(권장).** 조교가 강사에게 받은 자료나 홈페이지에서 직접
복사·저장한 텍스트를 파일로 넘깁니다. `.txt`, `.md`, `.pdf`, `.docx` 를 지원합니다.
```
python3 make_manuscript.py "강의.pdf" --vision \
  --ref-file "강사_소개.txt" "커리큘럼.pdf" "작가노트.docx"
```

**방법 2 — URL로 넣기.** 강사 홈페이지·전시 소개 페이지 URL을 그대로 넘기면
페이지 본문 텍스트를 뽑아 씁니다. 사이트 구조에 따라 광고나 메뉴가 함께
딸려 올 수 있어, 결과가 늘 깔끔하지는 않습니다.
```
python3 make_manuscript.py "강의.pdf" --vision \
  --ref-url "https://example.com/artist" "https://example.com/course"
```

두 옵션은 함께 써도 되고, 각각 여러 개를 나열해도 됩니다. 원고는 슬라이드와
참고 자료에 있는 사실만 근거로 씁니다(**참고 자료에 없는 사실은 지어내지 않습니다**).
자료가 너무 크면 프롬프트가 지나치게 무거워지므로, 자료 하나당 최대 약 2만 자까지만
반영되고 그 이상은 자동으로 잘립니다.

> 참고 자료를 넣으면 **API 호출당 입력 토큰이 커져 비용이 늘어납니다.** 꼭 필요한
> 자료만, 핵심 부분만 골라 넣는 편이 좋습니다. URL 스크래핑이 실패해도 프로그램은
> 그 자료만 건너뛰고 계속 진행합니다.

> 💡 **실전 팁 — 예술 강사 케이스.** 예술 강사는 대부분 `--vision` 과 `--ref-file`(또는
> `--ref-url`) 을 함께 켜게 되는데, **두 옵션을 다 켜면 토큰·시간 비용이 꽤 올라갑니다.**
> 그래서 참고 자료는 **강사 홈페이지에서 필요한 텍스트만 골라 `.txt`로 저장해 넘기는
> 방식**이 결과 품질도, 비용도 가장 안정적입니다. URL 방식은 사이트 광고·메뉴가 함께
> 딸려 오는 경우가 있어 **백업 수단으로** 두세요.

전체 옵션 보기: `python3 make_manuscript.py --help`

### 다른 강의를 새로 변환할 때
기존 파일을 지울 필요가 없습니다. **새 PDF를 폴더에 넣고, 출력 이름(`-o`)만 다르게**
주면 됩니다. 설치(2~3단계)와 키 설정(5단계)은 다시 할 필요 없이 6단계만 반복하면 됩니다.
```
python3 make_manuscript.py "2주차.pdf" -o "2주차_원고.docx"
```
(같은 `-o` 이름을 쓰면 이전 원고를 덮어쓰니, 강의마다 다른 이름을 권장합니다.)

### (선택) 질문 없이 옵션으로 분량 지정하기
보통은 위처럼 실행 시 뜨는 질문에 답하면 됩니다. 매번 묻지 않고 바로 지정하려면
`--min-pages`(최소 보장 쪽수)를 주세요. 장 수는 분량에 맞춰 자동으로 늘어납니다.

| 과제 기준 | 명령 예시 |
|---|---|
| 1강 · 내부 (35장 이상) | `python3 make_manuscript.py "강의.pdf" --min-pages 35` |
| 2강 · 내부 (70장 이상) | `python3 make_manuscript.py "강의.pdf" --min-pages 70` |
| 1강 · 외부 (17장 이상) | `python3 make_manuscript.py "강의.pdf" --min-pages 17` |
| 2강 · 외부 (34장 이상) | `python3 make_manuscript.py "강의.pdf" --min-pages 34` |

> 분량은 **지정한(또는 선택한) 최소 쪽수 이상이 되도록** 프로그램이 자동으로 맞춥니다
> (부족하면 내용을 더 전개해 채움).
>
> ※ 실제 워드 쪽수는 **컴퓨터·설치된 폰트에 따라 1~2쪽 정도 달라질 수 있습니다.**
> 안전하게 최소치를 넘기도록 약간 여유를 두고 작성합니다(예: 70장 요청 시 약 72쪽).
>
> ⚠️ **다만 슬라이드 글자가 적을수록 한계가 있습니다.** 텍스트가 적은 PPT를 큰 분량으로
> 늘리면, 같은 내용을 풀어 쓰거나 늘어지는 부분이 많아져 품질이 떨어집니다. 큰 분량
> 과제(내부 70장 등)는 **생성 후 사람이 반드시 검토·보완**하고, 가능하면 슬라이드 원문이
> 풍부한 자료에 쓰는 것이 좋습니다. (큰 목표일수록 모델 호출이 많아져 비용도 늘어납니다.)

## 문제가 생기면 (자주 나오는 오류)

| 화면에 나오는 말 | 뜻과 해결 |
|---|---|
| `command not found: python3` | 파이썬이 안 깔렸거나 PATH 미설정. 1단계 다시(윈도우는 "Add to PATH" 체크). |
| `No module named ...` | 3단계(`pip install -r requirements.txt`)를 안 했거나 다른 폴더에서 실행. 폴더 이동 후 다시. |
| `API 키가 없습니다` / 키를 물어봄 | 5단계 **방법 A**처럼 번호 누르고 키를 붙여넣으면 됩니다. 또는 `key.txt` 가 프로그램 폴더에 있는지 확인. |
| (맥) `'.'으로 시작하는 이름은 사용할 수 없습니다` | 맥은 `.env` 같은 점 파일을 못 만듭니다. **`.env` 대신 `key.txt`** 를 쓰거나, 5단계 방법 A(자동 입력)를 쓰세요. |
| `key.txt.example` 이 폴더에 안 보임 | 맥 Finder에서 `⌘+Shift+.`(점) 을 눌러 숨김 파일 표시. 또는 그냥 방법 A 사용. |
| `이미지/스캔 PDF로 의심됩니다` | 텍스트가 적은 PDF입니다. **`--vision`** 을 붙이면 그림·도표까지 읽어 처리합니다. |
| `입력 파일을 찾을 수 없습니다` | PDF가 폴더 안에 있는지, 따옴표 안 이름이 정확한지 확인. |

## 꼭 기억할 점

- 생성된 원고는 **초안**입니다. **반드시 사람이 한 번 읽고 검토**한 뒤 사용하세요.
  슬라이드(및 넣어 준 참고 자료)에 있는 내용만 근거로 쓰므로, 자료가 부실하면
  원고도 부실합니다.
- 수식·다이어그램·표·그림은 기본적으로 텍스트로 바뀌는 범위에서만 반영됩니다.
  이런 시각 자료가 많은 강의는 `--vision` 을 붙이면 그 내용까지 읽어 반영합니다.
- 예술 강사처럼 슬라이드가 이미지 위주여서 원고에 살이 안 붙는 경우,
  `--ref-file`/`--ref-url` 로 강사 홈페이지·수업 소개 등을 함께 넣어 근거를 보강할 수 있습니다.
- API 키와 슬라이드 PDF는 외부에 공유하지 마세요.
